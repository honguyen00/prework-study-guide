HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The header element appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • When using headings, it is important that we use heading tags in order from 1 to 6. In addition, the biggest heading, h1 should only appear once on each page.
  • The img element has the 'src' attribute defines the location, or source, of the image file and the 'alt' attrubute contains a text string that describes the appearance and functionality of the image.
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • A margin indicates how much space we want aroung the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • A link element creates the connection to external file. The rel attribute specifies the relationship between the current document and the linked document. The href attribute specifies the location (URL) of the exoernal resource
  • Assigning colors in CSS can be done by semantic references or hexadicimal code

Git

  • git config --global user.name "[name]": set a name that is identifiable for credit when review version history
  • git config --global user.email "[valid-email]": set an email address that will be associated with each history marker
  • git init: initialize an existing directory as a Git repository
  • git clone [url]: retrieve and entire repository from a hosed location via URL
  • git status: chechs what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: fetch and merge any commts from the tracking remote branch
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a set of instructions that tells the computer how to perform a certain task. A function is only executed when called
